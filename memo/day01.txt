웹 서버 (HTTP Web Server -> WS)
	- 클라이언트가 보낸 요청을 받아 처리한다.
	- 요청이 정적 컨텐츠를 요구하는지 동적 컨텐츠를 요구하는지 구분한다.
	- 정적인 컨텐츠를 요청한다면 미리 준비한 웹 문서를 response에 담아 보내주지만
	  동적인 컨텐츠를 처리할 수 없다.
	  즉, 웹서버는 정적인 페이지나 컨텐츠를 처리하는 서버이다.
	- 자바에서 사용하는 대표적인 웹 서버는 Apache가 있다.
	

WAS(Web Application Server)
	- 웹 서버와 동일하게 클라이언트의 요청을 받아 처리하는 서버이다.
	- WAS는 웹 컨테이너 또는 서블릿 컨테이너 라고도 불린다.
	- 웹 서버와 다르게 정적, 동적 컨텐츠를 모두 처리할 수 있다.
	  단, 정적 컨텐츠 처리는 웹 서버에 비해 성능이 떨어진다.
	- 동적 컨텐츠를 처리하기 위해 여러 연산 및 DB접근을 수행하며 프로그래밍 언어
	  를 활용한다.
	- JAVA에서 사용하는 대표적인 WAS는 Tomcat 이 있다.

일반적으로 웹 어플리케이션을 운영할 때 웹 서버와 WAS를 같이 사용하여 정적 컨텐츠와 동적 컨텐츠를
분리하여 처리한다.

** 아파치 톰캣
아파치 톰캣은 WAS로 분류되지만 Apache와 Tomcat을 합쳐놓은 형태이다.
즉, 아파치 톰캣은 내부에 WebServer를 포함하고 있다.
